# UI Style Preservation

- Preserve layout hierarchy, visual density, component relationships, and interaction flow during UI migration.
- Do not redesign the screen just because the target platform has different primitives.
- Keep information architecture stable unless the task or source behavior clearly requires a change.
- When a component cannot map one-to-one, preserve its interaction intent and user-facing state.
- Reuse target-platform conventions only where they support the original UI behavior.
