# Migration General

- Read or search the relevant source and target code before deciding how to change it.
- Preserve the source project's observable behavior unless the task explicitly asks for a change.
- Work in small steps and keep each edit focused on the current migration goal.
- Prefer existing target-project patterns over inventing new structure.
- After modifying files, inspect the current diff before claiming the work is complete.
- If build or test feedback fails, read the short failure summary before making the next small repair.
